Hoy en día es habitual navegar en dispositivos de distintos tamaños. La forma de resolver el problema es el Responsive Design. Funciona de la siguiente manera: el servidor envía CSS para cada uno de los dispositivos posibles y es el navegador el que filtra para saber qué reglas debe usar con cada dispositivo concreto.

La forma de separar las reglas CSS unas de otras para usar con distintos dispositivos es la media query

Se pueden meter dentro de una etiqueta link de los de la cabecera que usamos normalmente para el CSS pero lo normal es poner todas las reglas en el mismo fichero

	/*
	Esta zona no va protegida por ninguna media query y las reglas se aplican a todas las dimensiones
	*/
	body {background-color:#b6e9e8;}
	
	@media screen and (max-width: 700px){
	/*
	Las reglas que pongamos aquí sólo se usan si el dispositivo tiene menos de 700px
	*/
		body {background-color:#aedfae;}
	}
	
	@media screen and (max-width: 400px){
	/*
	Las reglas que pongamos aquí sólo se usan si el dispositivo tiene menos de 400px
	*/
		body {background-color:yellow;}
	}

A los valores 400px y 700px se les llama puntos de ruptura o breakpoints. Para ver cuáles usamos es razonable buscar en google qué es lo que se está usando en cada momento.

Normalmente el término no se traduce, pero si lo haces no le llames diseño adaptable. En inglés se distingue entre Responsive design, en el que se manda TODO EL CSS y es el navegador el que filtra y el Adaptative Design en el que el servidor manda un CSS u otro dependiendo del dispositivo que haga la petición.

No se puede llamar responsive design o responsive layout a esos diseños que hemos visto que se adaptan a distintas resoluciones (columna izquierda de ancho fijo y el resto lo ocupa la otra columna). A eso se le llama diseño fluido o líquido y el CSS es siempre el mismo
